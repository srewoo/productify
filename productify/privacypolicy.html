Productify · Privacy Policy

Privacy Policy

This Privacy Policy explains how the Productify browser extension handles your data. Productify is designed to keep as much data as possible on your device and to give you control over what is sent to external services.

What the extension stores locally

The extension stores the following data only in your browser's local storage:

  • Settings such as chosen STT provider, language, tone, output format, theme, and backend URL.
  • History of transcripts and generated outputs (so you can restore past runs).
  • Prompt library items you star/save.
  • Encrypted API keys for third‑party services (e.g. OpenAI, Google STT, ElevenLabs, Notion, GitHub, Jira, Linear, Slack, webhook URL).

You can erase all local data at any time from Settings → Data & Privacy → Clear All Local Data.

Microphone and audio data

  • Productify accesses your microphone only when you explicitly start a recording.
  • Audio is captured in an offscreen document, converted to an audio file, and sent to your configured backend server for transcription.
  • No raw audio is permanently stored by the extension; only the resulting text transcript is kept in history.

Data sent to your backend and AI providers

When you use Productify, the extension sends the following to your configured backend URL:

  • The audio recording (for Speech‑to‑Text).
  • The transcript text.
  • Optional page context, such as page title, URL, selected text, visible text, code blocks, headings, and limited structured data.
  • Your chosen intent (e.g. summarize, documentation, testing) and output preferences.

The backend then forwards data to providers you configure, such as OpenAI, Google Speech‑to‑Text, or ElevenLabs, using your API keys. Those requests are made directly from the backend server that you run and control.

API keys and credentials

  • API keys you enter are encrypted and stored in Chrome local storage; they are not transmitted to any third party except the backend you configure.
  • The backend uses your keys only to call the services you enabled (for example, OpenAI, Google STT, ElevenLabs, Notion, GitHub, Jira, Linear, Slack).
  • You can clear stored keys by clearing data from the extension or by resetting Chrome's site data for the extension.

Third‑party services

Productify integrates with external APIs such as OpenAI, Google Cloud, ElevenLabs, and productivity tools. Your use of those services is governed by their own terms and privacy policies. You are responsible for complying with those policies when providing keys and sending data.

Data retention

  • History and library entries remain in your browser until you delete them or use the clear‑data option.
  • The backend and third‑party providers may retain logs or content according to their own retention settings and configurations.

Security

  • The extension uses browser storage and the Web Crypto API to encrypt API keys before storing them.
  • No credentials are hard‑coded into the extension source; you provide and control all keys.
  • For production use, you should run the backend over HTTPS and secure server access appropriately.

Your choices

  • You can revoke microphone access at any time via your browser settings.
  • You can clear all local extension data at any time from the Settings page.
  • You can choose which providers to enable by adding or omitting API keys.

Contact

If you have questions about how Productify handles data, please contact the maintainer of this extension or review the project README in the corresponding source repository.
