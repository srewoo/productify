Using Productify

Productify is a voice‑first AI side panel for your browser. Speak your ideas, tasks, or requests and it turns them into structured AI output you can copy, refine, and send into your tools.

Basic flow

1. Open the side panel – click the Productify icon or use the keyboard shortcut defined in Chrome.

2. Record – click the mic button or hold Space while the panel is focused, then speak.

3. Transcribe – audio is sent to your selected Speech‑to‑Text provider (OpenAI, Google, or ElevenLabs).

4. Generate – the transcript plus page context is sent to the backend, which uses GPT‑4o to produce the result.

5. Use the output – copy it, refine it, export it, or send it to Notion, GitHub, Jira, Linear, Slack, or a webhook.

Turning ideas into prompts and specs

You can open Productify on any page (even an empty one), click the mic, and talk through what you want to build. Productify will:

  • Transcribe your speech into text.
  • Classify your intent (for example prompt generation, task extraction, user story, or documentation).
  • Generate a structured output you can use as a prompt, spec, or backlog.

For long ideas, it works best to record in shorter segments (30–90 seconds) and use the Refine controls to iterate.

History and prompt library

  • Every run is saved to History in the main view.
  • You can restore a past item to bring back its transcript, intent, and output.
  • Use the ⭐ button to add important outputs to the Prompt Library for quick reuse.

Integrations

In Settings → Integrations you can provide API keys or URLs for Notion, GitHub, Jira, Linear, Slack, or a custom webhook. Once configured, the Send button lets you route the latest output into those tools.

Microphone and permissions

  • Chrome will ask for microphone permission the first time you record. Choose Allow.
  • If you blocked it earlier, go to chrome://settings/content/microphone and allow the Productify extension.
  • You can clear all local data (history, settings, keys) from Settings → Data & Privacy → Clear All Local Data.

Troubleshooting

  • “Microphone Access Denied” – check Chrome’s mic settings and macOS Privacy & Security → Microphone.
  • “Processing Failed” – verify your backend URL and OpenAI key in Settings → API Keys and make sure the backend server is running.
  • No output – ensure your STT provider key is set (Google / ElevenLabs) or Whisper is allowed for your OpenAI project.
