Briefly · Privacy Policy

Privacy Policy

This Privacy Policy explains how the Briefly browser extension handles your data. Briefly is designed to keep as much data as possible on your device and to give you control over what is sent to external services.

What the extension stores locally

The extension stores the following data only in your browser's local storage:

  • Settings such as chosen STT provider, language, tone, output format, and theme.
  • Workflow preferences such as the active recipe, page-context usage, selection-only mode, and redaction settings.
  • History of transcripts and generated outputs (so you can restore past runs).
  • Prompt library items you star/save.
  • Encrypted API keys for third‑party services (e.g. OpenAI, Google STT, ElevenLabs, Notion, GitHub, Jira, Linear, Slack, webhook URL).

You can erase all local data at any time from Settings → Data & Privacy → Clear All Local Data.

Microphone and audio data

  • Briefly accesses your microphone only when you explicitly start a recording.
  • Audio is captured in an offscreen document, converted to an audio file, and sent directly from the extension to the Speech‑to‑Text provider you configured (OpenAI, Google, or ElevenLabs).
  • No raw audio is permanently stored by the extension; only the resulting text transcript is kept in history.

Data sent to AI providers

When you use Briefly, the extension sends the following to the AI APIs you configure:

  • The audio recording (for Speech‑to‑Text).
  • The transcript text.
  • Any typed request you enter directly into the command composer.
  • Optional page context, such as page title, URL, selected text, visible text, code blocks, headings, and limited structured data.
  • Your chosen intent (e.g. summarize, documentation, testing) and output preferences.

These requests are made directly from your browser to providers such as OpenAI, Google Speech‑to‑Text, or ElevenLabs using API keys you supply in the extension settings.

If redaction is enabled, Briefly attempts to remove common secrets such as API keys, tokens, email addresses, and long numeric strings before sending page context for generation.

API keys and credentials

  • API keys you enter are encrypted and stored in Chrome local storage; they are only used by the extension to call the services you configure.
  • The extension uses your keys only to call selected providers (for example, OpenAI, Google STT, ElevenLabs, Notion, GitHub, Jira, Linear, Slack).
  • You can clear stored keys by clearing data from the extension or by resetting Chrome's site data for the extension.

Third‑party services

Briefly integrates with external APIs such as OpenAI, Google Cloud, ElevenLabs, and productivity tools. Your use of those services is governed by their own terms and privacy policies. You are responsible for complying with those policies when providing keys and sending data.

Data retention

  • History and library entries remain in your browser until you delete them or use the clear‑data option.
  • Third‑party providers may retain logs or content according to their own retention settings and configurations.

Security

  • The extension uses browser storage and the Web Crypto API to encrypt API keys before storing them.
  • No credentials are hard‑coded into the extension source; you provide and control all keys.
  • You should keep your API keys secret and rotate them regularly following provider best practices.

Your choices

  • You can revoke microphone access at any time via your browser settings.
  • You can clear all local extension data at any time from the Settings page.
  • You can choose which providers to enable by adding or omitting API keys.

Contact

If you have questions about how Briefly handles data, please contact the maintainer of this extension or review the project README in the corresponding source repository.
